# SQL-Based Business Analytics & KPI Analysis

## Overview

This project uses SQL to analyze customer transaction data and extract key business metrics (KPIs). The goal is to demonstrate how SQL can be used to support data-driven decision-making by identifying patterns in revenue, customer behavior, and performance across different segments.

## Objectives

* Analyze transactional data using SQL
* Calculate key business metrics (KPIs)
* Identify trends in revenue and customer behavior
* Translate data findings into actionable insights

## Dataset

The dataset contains customer transaction information, including:

* Total bill amount
* Tip amount
* Day and time of transaction
* Customer attributes (e.g., smoker, group size)

## Key Analysis

### Total Revenue

Calculated overall revenue generated from all transactions.

### Average Revenue

Measured average transaction value to understand typical customer spending.

### Revenue by Day

Analyzed which days generate the most revenue.

### Customer Segmentation

Examined how group size and customer type impact spending behavior.

## Key Insights

* Revenue is unevenly distributed across different days, indicating that certain periods drive more business activity
* Larger group sizes tend to generate higher average spending
* Specific customer segments contribute more significantly to overall revenue

## Business Impact

* High-performing days should be prioritized for staffing and planning
* High-value customer segments should be targeted for retention
* Data-driven insights can improve operational and strategic decisions

## Tools Used

* Python
* Pandas
* SQL (via pandasql)
* Matplotlib / Seaborn

## Conclusion

This project demonstrates how SQL can be used to extract meaningful business insights from data. By analyzing key metrics and customer behavior, businesses can make more informed decisions and improve performance.
